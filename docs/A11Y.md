# Accessibility Testing Guide

## Tools Available

### 1. **axe DevTools** (Chrome/Firefox Extension)
- Most detailed violations
- Browser DevTools integration
- Free, premium version with more features

Install: https://www.deque.com/axe/devtools/

### 2. **WAVE** (Chrome/Firefox Extension)
- Visual feedback on page
- Contrast checker
- Heading structure

Install: https://wave.webaim.org/extension/

### 3. **Pa11y CI** (Automated)
Test all apps automatically:

```bash
# Single run
pnpm a11y

# With CSV export
pnpm a11y:report

# Using helper script
./scripts/test-a11y.sh
```

### 4. **Lighthouse DevTools** (Chrome native)
Run audit in DevTools → Lighthouse tab
- Includes accessibility score
- Performance + SEO combined

### 5. **VoiceOver / NVDA** (Screen readers)
- **Mac**: Cmd+F5 (VoiceOver)
- **Windows**: NVDA (free) https://www.nvaccess.org/

Test keyboard navigation:
- Tab through all interactive elements
- Enter/Space activation
- Arrow keys in lists/menus

## Standard: WCAG 2.0 Level AA

All tests use **WCAG2AA** standard (industry baseline).

## Configuration

Pa11y config: `.pa11yci.json`
- Tests: host (3000), home (3001), media (3002)
- Standard: WCAG2AA
- Level: error

## Workflow

1. **Local dev**: axe DevTools + WAVE
2. **Before commit**: `pnpm a11y`
3. **Spot check**: VoiceOver/NVDA (keyboard + screen reader)
4. **CI/CD**: Automated `pnpm a11y` (planned)

## Common Issues & Fixes

| Issue | Cause | Fix |
|-------|-------|-----|
| Contrast fail | Text too light on background | Use darker token (e.g., `text-badge-foreground`) |
| Missing alt text | Images without `alt` | Add meaningful `alt="description"` |
| Link without name | No text in `<a>` | Add text or `aria-label` |
| Form label mismatch | Label text ≠ input name | Use `<label htmlFor="id">` |
| Heading order broken | Skip heading levels | Use h1→h2→h3 sequentially |
| Button too small | Touch target < 48x48px | Increase padding/size |
